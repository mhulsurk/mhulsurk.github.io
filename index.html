Steps of LAXPC data analysis using LAXPC software (Format A) available at link http://astrosat-ssc.iucaa.in/?q=laxpcData


Software having individual routines to extract spectra,light curves, power spectra, energy and frequency dependent time-lag, Dynamic power spectra, flux resolved spectroscopy, faint source background estimation (May 19 2018 version). Prior to software installation it it mandatory to install heasoft in your system (https://heasarc.gsfc.nasa.gov/docs/software.html).


  • Configuring the software [README]
    1. Create a Directory in your home (or anywhere else) called LAXPCsoftware
    2. untar the tar file 'laxpcsoft.tar.gz' in the LAXPCsoftware directory e.g. /home/yourname/LAXPCsoftware/
    3. Put the following commands in the .bashrc file located at you home directory
      export LAXPCSOFT="/home/yourname/LAXPCsoftware/"
      export LAXPCPATH="$LAXPCSOFT/LAXPC_CAL/"
      export PATH="$PATH:$LAXPCSOFT/laxpc_bin"
    4. Open a new terminal
    5. In LAXPCsoftware run the command :
      ./Configure
      (A succesful configuration will give no errors or statements.)
      But if there is error statement then use command :
      ./Configure_curl

  • Data Analysis Procedure

    You can download publicly available LAXPC data from the webpage
    https://astrobrowse.issdc.gov.in/astro_archive/archive/Search-Record.jsp
    Sample source is crab data (observation id : 9000000778, observation date : 8th November 2019)

    1. Create a data folder
      User should create a data folder to store unziped level 1 laxpc data
      >mkdir laxpc_data
      Path of data folder is : /home/user_name/walkthrough/LAXPC/laxpc_data
      >cd /home/user_name/walkthrough/LAXPC/laxpc_data
      > ls
      LEVL1AS1LXP20161108A02_090T01_9000000778_06050.zip LEVL1AS1LXP20161108A02_090T01_9000000778_06051.zip
      This will create a folder 20161108_A02_090T01_9000000778_level1.
    2. Unzip level 1 data files
      >unzip LEVL1AS1LXP20161108A02_090T01_9000000778_06050.zip
      >unzip LEVL1AS1LXP20161108A02_090T01_9000000778_06051.zip
      INTIALIZE HEASOFT (heainit in the terminal)
    3. Now create an analysis folder (working directory) [README]
      >mkdir analysis_laxpc
      Path of analysis directory is : /home/user_name/walkthrough/LAXPC/analysis_laxpc
    4. Set the data path in the working directory [README]
      In the analysis_laxpc folder (Path: /home/user_name/walkthrough/LAXPC/analysis_laxpc)
      set the variable $LAXPCDATAPATH to where your data is ending with the laxpc directory.
      Example :
      > export LAXPCDATAPATH="/home/user_name/walkthrough/LAXPC/laxpc_data/ 20161108_A02_090T01_9000000778_level1/laxpc/”
    5. Make data file lists [README]
      Data file lists of level 1 data sets and the mkf file of each orbit may be listed in a txt file by giving the command
      > laxpc_make_filelist
      this will produce two text files "eventfiles" and "filterfiles".
    6. Check the content of eventfiles and filterfiles
      Check the content of eventfiles and filterfiles for appropriate path of data and .mkf files.
      To check the content of the files type in terminal
      >more eventfile
      (eventfile is the list of all the level 1 event files of laxpc 10, 20 and 30.)
      >more filterfiles.
      (filterfiles is the list of all the level 1 .mkf files which stores the housekeeping information of each orbit file)
      Example:
      >more eventfiles
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06050/lxp1/modeEA/AS1A02_090T01_9000000778lxp1EA_level1.fits"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06050/lxp2/modeEA/AS1A02_090T01_9000000778lxp2EA_level1.fits"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06050/lxp3/modeEA/AS1A02_090T01_9000000778lxp3EA_level1.fits"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06051/lxp1/modeEA/AS1A02_090T01_9000000778lxp1EA_level1.fits"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06051/lxp2/modeEA/AS1A02_090T01_9000000778lxp2EA_level1.fits"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06051/lxp3/modeEA/AS1A02_090T01_9000000778lxp3EA_level1.fits"
      >more filterfiles
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06050/AS1A02_090T01_9000000778lxp_level1.mkf"
      "/home/user_name/walkthrough/LAXPC/laxpc_data/20161108_A02_090T01_9000000778_level1/laxpc//06051/AS1A02_090T01_9000000778lxp_level1.mkf"
    7. Make FITS event file [README]
      To generate merged level2 event fits file from level 1 fits orbit file list (eventfiles) use the command laxpc_make_event
      >laxpc_make_event eventfiles
      this will produce a file called 'level2.event.fits'.
      The level 2 event output file, level2.event.fits, is a fits file contains information of time of arrival, channel number, layer number of a laxpc unit, laxpc unit number and energy information of each X-ray photon.
    8. Make gti file [README]
      >laxpc_make_stdgti filterfiles
      This command will produce a file called 'usergti' which has the GOOD TIME INTERVALS
      (GTI)which removes earth occulation and SAA.
      (Note: "laxpc_make_occultgti filterfiles" will instead make occultgti
      having the time when the source was occulted).
      > laxpc_fluxresl_gti [README_FLUXRESLGTI]
      Usage: laxpc_fluxresl_gti -l ratemin -h ratemax -o output_gti_filename filename.lc
      Flux resolved gti can also be created using the command. This task produces a GTI file corresponding to the times when the rate in the lightcurve was between a minimum value (ratemin) and maximum value (ratemax).
      The output gti can then be used to generate spectrum and other science products as described in the following analysis steps.
    9. To make source spectrum [README_SPEC]
      Usage: laxpc_make_spectra [-u (user gti asci file)] [-r resp_files] [-l Layer no] [-g Energy grouping factor] level2.event.fits
      Here, flag l is for the layer input. Options are either all or 1 which, means either all the 5 layers of LAXPC will be used for spectrum extraction or the layer 1 (Anode 1 and 2) will be used for spectrum extraction.
      If Layer 1 has been used as filter, then the files will have an additional _L1 in the filenames.
      The correct responses will be picked up by the code in Default mode.
      Example:
      > laxpc_make_spectra -u usergti.fits level2.event.fits
      This command will generate spectrum of the source (example:spectrum_20.pha ) and grouped spectrum (example: spectrum_grp_20.pha) of source. This will automatically select appropriate rmf (example: lx20cshm04v1.0.rmf) files for the data sets for each detector and copy it in current directory.
    10. To generate laxpc background spectrum [README_BACKSPEC]
      Usage: laxpc_make_backspectra -u (user gti file in fits format) -g gain -l layer -s syserr
      Prior to running this command it is MUST obtain source spectra (using command laxpc_make_spectra) first for the SAME usergti (good time interval file) and layer flags.
      Example:
      >laxpc_make_backspectra -u usergti.fits filterfiles
      This command will generate simulated spectrum of the background (example: backlxp20.pha) for each detectors.
    11. Create an energy input flle
      User may create a energy input file for example eneinput and save the energy ranges as per science interest.
      >vi eneinput
      Example:
      Save two energy ranges in the file 3-5 keV and 5-20 keV
      3.0 5.0
      5.0 20.0
      Note : you can define upto nine energy range in incremental order. The energy bands should not overlap i.e. 3-15 and 10-30
    12. To Make Lightcurves [README_LC]
      Usage: laxpc_make_lightcurve [-p (which pcu? all or e.g. 12)] [-t timebin] [-u (user gti asci file)] [-o output filename] [-e energy define file] [-l Layer no] level2.event.fits
      Example:
      To create lightcurves in 3-5keV and 5-20 keV energy bands, combining LAXPC 10 and 20, applying usergti file and 1 sec binning:
      >laxpc_make_lightcurve -p 12 -t 1.0 -u usergti.fits -e eneinput level2.event.fits
      Ouput files: lightcurve_3.0_5.0keV.lc and lightcurve_5.0_20.0keV.lc
      To create lightcurve from top layer of combined LAXPC 10 and 20 then the above command may be modified as:
      >laxpc_make_lightcurve -p 12 -t 1.0 -u usergti.fits -e eneinput -l 1 level2.event.fits
      Ouput files: lightcurve_3.0_5.0keV_L1.lc and lightcurve_5.0_20.0keV_L1.lc
      (User can plot the lightcurves using ftools fv, fplot or lcurve in heasoft.)
    13. To Make Background Lightcurves [README_BACKLC]
      Usage: laxpc_make_backlightcurve -p (which pcu? all or e.g. 12) -t timebin -u user_gti_file -e energy define file -l Layer no -s syserr filterfiles
      To run this command you MUST obtain source and background spectra first for the SAME usergti file and layer. Run laxpc_make_spectra and laxpc_find_back with the same -u and -l flags BEFORE running this command.
      Example:
      To generate background lightcurves in 3-5keV and 5-20 keV energy bands, combining LAXPC10 and 20, applying usergti file and 1 sec binning:
      >laxpc_make_backlightcurve -p 12 -t 1.0 -u usergti.fits -e eneinput filterfiles
      Ouput files: Back_lightcurve_3.0_5.0keV.lc and Back_lightcurve_5.0_20.0keV.lc
      The layer 1 LAXPC 10 or LAXPC 20 background for faint sources where the source contribution to the 50-80 keV count rate is less than 0.25 counts/sec (15 milli-crabs or 6 × 10−9 ergs/s/cm2). In such cases the observed 50-80 keV count rate can be used to estimate the background spectrum and lightcurve using the set of steps explained in the readme file [ README_BACKSPEC_F_L10 , README_BACKLCF_L20.]
      Note that the background lightcurve will always be of 100 second time bin.
    14. To generate energy dependent Time Lags [README_FREQLAG]
      Usage: laxpc_find_freqlag [-h (high frequency in Hz)] [-l (low frequency in Hz)] [-f (Frequency at which lags are to be computed in Hz)][-p (which pcu? all or e.g. 12)] [-u (user gti asci file)] [-o output filename] [-e energy define file] level2.events.fits
      Example:
      laxpc_find_freqlag -l 0.1 -h 500.0 -f 30.0 -p 12 -u usergti.fits -e eneinput level2.event.fits
      This command will search the range of frequency in the range : lower frequency 0.1 Hz to higher frequency 500 Hz around the peak frequency 30 Hz. To search time lag PCU 1 and 2 are used. The output file "NPow_outputfile” denotes the energy ranges used. For N=1, 2 File containing information for a particular energy band. For example If the selected energy range is 3-5 keV and 5-20 keV then 1Pow_outputfile denotes the power spectrum in 3-5 keV energy band and 2Pow_outputfile in 5-20 keV energy band. If there is no energy range input from the user then the default energy ranges are 3-15 and 15-30 keV.
      Different columns in the output file are:
      1 Mid Point Energy (keV) should be same for all rows
      2 Energy Range (keV) should be same for all rows
      3 Freq (Hz)
      4 Coherence
      5 Coherence error
      6 Time lag (secs)
      7 Error on time lag
      8 Power
      9 Error on Power
      10 Intrinsic coherence (i.e.dead time poisson fluction subtracted)
      11 Intrinsic Coherence error
      12 Poisson noise level for Power spectrum (Deadtime corrected)
    15. laxpc_rebin_power [README_rebin_power]
      Usage: laxpc_rebin_power [-s snr] [-h max freq bin] [-l min freq bin]
      Here flag s the signal to noise (SNR) ratio that the power spectra will be binned to. Default value is 5 snr. The flag h means the maximum frequency width which any bin will have irrespective of its SNR. Default value of flag h is 2.0. The flag l means the minimum frequency width which any bin will have irrespective of its SNR. Default value of flag l is 1.05.
      This command helps in rebining power density spectra (PDS) in frequency domain. Further it converts the PDS in equivalent .pha files compatible with XSPEC and allows the PDS to be fitted using XSPEC. >laxpc_rebin_power powfiles
      Name of file being used: powfiles
      if, N=2 (N is the number of energy range used in laxpc_find_freqlag to generate PDS)
      1Pow_level2.event
      2Pow_level2.event
      powfile file is the list of the *Pow files created by running the time lag command laxpc_find_freqlag in the previous step.
      Output files: reb_backPow_level2.event.pha1 reb_backPow_level2.event.pha2, reb_pha1Pow_level2.event.rsp, reb_pha2Pow_level2.event.rsp, reb_Pow_level2.event.pha1, reb_Pow_level2.event.pha2
